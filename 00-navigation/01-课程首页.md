# Agent 设计与上下文工程

> [!note] 知识库说明
> **定位**：这不是 API 教程，也不是框架使用手册，而是一套面向**有 Agent 基础开发者**的系统化知识库。
> **目标**：帮你建立稳定的 Agent 设计语言，理解上下文工程的边界、方法与系统约束。
> **结构原则**：每章优先讲通用原理，再做跨场景映射，最后回到功能追齐 Agent 这个主案例。
> **贯穿案例**：功能追齐 Agent——将 H5 页面功能对齐到 App 最新版本，用来承接每章的深度落地。
> **阅读方式**：优先走主线，再按问题索引和学习路径跳转补洞。

---

> [!question] 核心问题
> 当我们说"设计一个 Agent 系统"时，我们究竟在设计什么？
> 是一个 Prompt、一组工具，还是由目标、上下文、状态、记忆、执行与评测闭环构成的运行系统？

## 这套知识库要解决的错觉

> [!warning] 误区
> - 把 Agent 的所有问题都理解成 Prompt 问题
> - 把长上下文误当成好上下文
> - 把多 Agent 误当成更高级的架构
> - 把"能跑"误当成"可设计、可评估、可维护"
