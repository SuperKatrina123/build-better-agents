# 学习路径

> [!abstract] 文档定位
> 这页不解释概念，只回答一个问题：面对这套知识库，你该从哪里开始读。
> 根据你的背景和目标，选一条路径；如果你不是在做 Coding Agent，优先结合跨场景迁移指南一起读。

---

## 路径 A：建立完整系统认知

**适合**：做过 Agent Demo，但对"为什么不稳定、为什么出错"仍然没有统一解释

```
01 Agent认知基础
  → 02 上下文工程（重点：2.1~2.3）
  → 03 工具设计 ACI
  → 04 记忆系统
  → 06 Harness
  → 08 评测体系
```

**核心收获**：建立从上下文 → 工具 → 记忆 → 安全控制的完整设计语言

---

## 路径 B：上下文工程专项

**适合**：LLM 推理结果不稳定，想从根本上理解上下文管理

```
02.1 Context Rot（必读）
  → 02.2 五层上下文模型
  → 02.3 Skills 按需加载
  → 02.4 三种压缩策略
  → 02.5 Prompt Caching
  → 02.7 设计决策 2（落地）
```

**核心收获**：能设计上下文预算方案，让 LLM 始终"看到最重要的信息"

---

## 路径 C：工具系统专项

**适合**：工具调用经常失败或结果不对，想重新设计工具接口

```
03.1 什么是 ACI
  → 03.2 输入设计
  → 03.3 输出设计
  → 03.4 工具粒度
  → 03.5 工具错误处理
  → 03.6 设计决策 3（落地）
```

**核心收获**：能按 ACI 原则设计让 LLM 真正用好的工具接口

---

## 路径 D：长任务与多 Agent 专项

**适合**：任务复杂度超过单次会话，或正在评估是否需要多 Agent

```
05 规划与长任务（全章）
  → 07.1 为什么需要多 Agent
  → 07.2 多 Agent 拓扑模式
  → 07.3 Orchestrator 设计
  → 07.6 设计决策 7（落地）
```

**核心收获**：能设计任务分解 + 检查点 + Orchestrator 的完整长任务方案

---

## 路径 E：生产化专项

**适合**：系统已经能跑，正在考虑上线，关心稳定性、成本和可维护性

```
06 Harness（全章）
  → 08 评测体系（全章）
  → 09.1 从原型到生产的差距
  → 09.2 可观测性设计
  → 09.3 成本控制与优化
  → 09.6 设计决策 9（生产化路径）
```

**核心收获**：能按三阶段路径把 Agent 从原型推向生产

---

## 路径 F：跨场景迁移专项

**适合**：你关心的不只是 Coding Agent，而是想把同一套方法迁移到 Research、客服或业务流程 Agent

```
00-navigation/05 跨场景迁移指南
  → 01 Agent认知基础
  → 02 上下文工程
  → 05 规划与长任务
  → 06 Harness
  → 08 评测体系
```

**核心收获**：能区分“哪些是所有 Agent 共享的稳定原则”，以及“哪些只是功能追齐 Agent 这个案例的具体实现”。
