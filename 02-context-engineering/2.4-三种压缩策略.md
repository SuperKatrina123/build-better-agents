# 2.4 三种压缩策略：上下文超长时怎么办

> [!abstract] 本节导读
> Skills 按需加载控制了"起点"——任务开始时上下文足够干净。
> 但任务执行过程中，对话流（L5）和工作记忆（L4）会持续增长。
> 这一节解决"运行中上下文超长"的问题，给出三种压缩策略和选择框架。
> **预计时长**：核心阅读 20 分钟。

---

> [!question] 带着问题阅读
> 一个追齐任务跑到第 15 轮，上下文里有大量工具返回的代码片段。
> 你会怎么处理它们？直接截断？还是有更好的方法？

---

## 1. 为什么需要主动压缩

即使做了 Skills 按需加载，L5 对话流仍然会增长：

- 每次 `read_repo_file` 返回几十行代码
- 每次 `diff_feature` 返回差异报告
- 每轮的 Assistant 回复也会保留在 messages[]

**不主动压缩的后果**：
- 超过模型的上下文窗口（通常 128K~200K tokens），直接报错
- 更早发生的问题：信噪比下降，Context Rot 发作（见 2.1）

**关键认知**：压缩不是"不得已而为之"，而是**主动的上下文治理**。
就像代码需要重构，上下文也需要定期整理。

---

## 2. 三种压缩策略

![[compression-strategies.excalidraw]]

### 策略一：截断（Truncation）

**做法**：丢弃最早的 N 轮对话，只保留最近 K 轮。

**触发时机**：对话轮次超过阈值（推荐：超过 10 轮时触发）

**实现**：
```typescript
function truncateMessages(messages: Message[], keepLast: number): Message[] {
  // 始终保留 system message
  const systemMsg = messages.filter(m => m.role === "system");
  const others = messages.filter(m => m.role !== "system");
  return [...systemMsg, ...others.slice(-keepLast)];
}
```

> [!tip] 原则
> 截断是最简单的策略，但有一个前提：**任务目标必须已经固化在 L1 System Prompt 里**。
> 如果关键约束只在某个早期轮次的对话里，截断会把它丢掉。
> 功能追齐 Agent 的追齐目标和规则都在 System Prompt，所以截断相对安全。

---

### 策略二：摘要（Summarization）

**做法**：把早期的 N 轮对话压缩成一个"历史摘要块"，替换原始对话。

**触发时机**：任务持续时间长（超过 20 轮），或早期结论很重要、不能丢弃。

**实现思路**：
```typescript
async function summarizeOldMessages(
  messages: Message[],
  keepRecentN: number,
  model: LLM
): Promise<Message[]> {
  const systemMsg = messages.filter(m => m.role === "system");
  const others = messages.filter(m => m.role !== "system");

  const toSummarize = others.slice(0, -keepRecentN);
  const toKeep = others.slice(-keepRecentN);

  const summary = await model.complete(`
    请将以下对话历史压缩为一段简洁的摘要，保留所有关键决策和结论，丢弃过程细节：
    ${JSON.stringify(toSummarize)}
  `);

  const summaryMessage: Message = {
    role: "user",
    content: `[历史摘要]\n${summary}\n[以上为之前的对话摘要，以下为近期对话]`
  };

  return [...systemMsg, summaryMessage, ...toKeep];
}
```

> [!warning] 误区
> **摘要压缩不是免费的**——它需要额外调用一次 LLM，产生延迟和 token 消耗。
> 而且摘要质量取决于模型，关键细节可能在压缩中失真。
> 所以不要频繁触发摘要，建议设定比截断更高的触发阈值（比如 20 轮）。

---

### 策略三：选择性保留（Selective Retention）

**做法**：给每一轮对话打"相关性分"，只保留高分轮次。

**触发时机**：超长任务（50+ 轮），工具输出内容差异很大，部分轮次与当前任务完全无关。

**实现思路**：
```typescript
function scoreMessageRelevance(
  message: Message,
  currentTask: string
): number {
  // 简单实现：关键词匹配
  const keywords = extractKeywords(currentTask);
  const content = JSON.stringify(message.content);
  const matches = keywords.filter(kw => content.includes(kw)).length;
  return matches / keywords.length;
}

function selectiveRetain(
  messages: Message[],
  currentTask: string,
  minScore: number = 0.3
): Message[] {
  const systemMsg = messages.filter(m => m.role === "system");
  const others = messages.filter(m => m.role !== "system");
  // 始终保留最近 5 轮
  const recentN = others.slice(-5);
  const older = others.slice(0, -5)
    .filter(m => scoreMessageRelevance(m, currentTask) >= minScore);
  return [...systemMsg, ...older, ...recentN];
}
```

## 3. 如何选择策略

| 场景 | 推荐策略 |
|------|---------|
| 短任务（< 10 轮），目标在 System Prompt | 截断 |
| 中等任务（10~30 轮），有重要历史决策 | 摘要 |
| 长任务（30+ 轮），工具输出差异大 | 选择性保留 |
| 不确定 | 先用截断，观察效果后再升级 |

> [!info] 方法
> **功能追齐 Agent 的推荐配置：**
> - L4 工作记忆（状态 JSON）：每轮覆盖式更新，不累积，本质上是最强的"压缩"
> - L5 对话流：超过 10 轮时触发截断，保留最近 8 轮
> - 追齐任务超过 25 轮时：对 10~25 轮的内容做摘要，保留最近 10 轮原文

## 4. 工具返回内容的预处理

压缩的另一个战场在**工具层**，而不只是 messages[]。

工具返回的内容进入对话流之前，应该先做预处理：

```typescript
// 在工具调用的 handler 里做预处理
async function readRepoFileHandler(repo: string, path: string): Promise<string> {
  const rawContent = await readFile(repo, path);

  // 文件超过 200 行，先摘要
  if (rawContent.split('\n').length > 200) {
    return await summarizeCode(rawContent, {
      keepImports: true,
      keepInterfaces: true,
      keepFunctionSignatures: true,
      collapseImplementation: true,
    });
  }

  return rawContent;
}
```

**代码文件压缩的实用规则**：
- 保留 import 声明（体现依赖关系）
- 保留 interface / type 定义（体现数据结构）
- 保留函数签名（体现 API）
- 折叠函数实现（除非与当前任务直接相关）

> [!example] 案例
> 购物车 H5 仓库的 `CartItemList.tsx` 有 320 行。
> 预处理后保留到约 60 行（imports + interfaces + 函数签名）。
> 追齐分析任务需要的信息全部保留，token 消耗降低到原来的 18%。

## 本节小结

- 三种策略：截断（简单快速）→ 摘要（保留脉络）→ 选择性保留（精准但复杂）
- 选择策略先看任务长度，再看历史决策的重要性
- 功能追齐 Agent：L4 覆盖式更新 + L5 超 10 轮截断 + 超 25 轮摘要
- 工具层预处理是另一个压缩战场：代码文件折叠实现，保留签名和接口
