# 9.2 可观测性设计：让 Agent 的行为可见

> [!abstract] 本节摘要
> 你无法改进你无法观察的东西。Agent 的可观测性包括三个层次：**日志**（发生了什么）、**追踪**（怎么发生的）、**指标**（发生了多少次 / 多快 / 多贵）。三者结合，才能让你在生产环境中真正掌握 Agent 的运行状态。

> [!question] 带着问题阅读
> - 如果你的 Agent 今天在生产中失败了，你需要多久才能定位是哪一步出了问题？
> - 你现在记录了哪些信息？哪些信息是你根本看不到的？

---

## 1. 三层可观测性

![[observability-stack.excalidraw]]

三层可观测性从下往上构建：

- **Agent Execution**（最底层）：实际执行的原始操作——工具调用、LLM 调用、记忆读写
- **Logs / Traces / Metrics**（中间层）：对原始操作的结构化记录和聚合
- **Dashboard + Alerts**（最上层）：把数据变成可视化和可操作的信号

### 日志（Logs）

日志回答"发生了什么"。追齐 Agent 需要记录：

- 每次工具调用（调用了什么工具、输入是什么、输出是什么、是否成功）
- 每次 LLM 调用（用了哪个模型、消耗了多少 token、耗时多少）
- 每次记忆读写（读了哪个 key、写入了什么、格式是否合法）

最简单的实现：**JSONL 文件**，每行一条记录，方便 `grep` 和脚本分析。

### 追踪（Traces）

追踪回答"怎么发生的"。它把一次任务执行的所有操作串联成一条链路：

```
session_id: abc-123
  step 1: read_memory → success (23ms)
  step 2: list_repo_files(repo=app-home) → success (156ms)
  step 3: LLM call (claude-sonnet) → 1847 tokens, success (2341ms)
  step 4: diff_feature(feature=dark-mode) → success (89ms)
  step 5: write_memory → success (12ms)
```

有了追踪，当第 3 步 LLM 调用失败时，你可以立刻知道前面的上下文是什么。

### 指标（Metrics）

指标回答"发生了多少次 / 多快 / 多贵"：

- 每日追齐功能数（throughput）
- 平均工具调用次数/功能（效率指标）
- 平均 token 消耗/功能（成本指标）
- 失败率（可靠性指标）
- P95 耗时（用户体验指标）

## 2. 结构化日志设计

### 日志字段定义

每条日志记录的标准字段：

```typescript
interface AgentLogEntry {
  timestamp: string;       // ISO 8601，如 "2024-01-15T10:23:45.123Z"
  session_id: string;      // 一次任务执行的唯一 ID，如 "align-20240115-abc123"
  agent_name: string;      // "orchestrator" | "research" | "analysis" | "execution"
  event_type: string;      // "tool_call" | "llm_call" | "memory_read" | "memory_write" | "task_start" | "task_end"
  tool_name?: string;      // 工具调用时填入，如 "read_repo_file"
  input_summary?: string;  // 输入的摘要（不要存完整内容，太大）
  output_summary?: string; // 输出的摘要
  token_count?: number;    // LLM 调用时的 token 消耗
  duration_ms: number;     // 操作耗时（毫秒）
  success: boolean;        // 是否成功
  error?: string;          // 失败时的错误信息
}
```

### 日志写入实现

```typescript
import * as fs from 'fs';
import * as path from 'path';

class AgentLogger {
  private logFile: string;

  constructor(sessionId: string, logDir = './logs') {
    fs.mkdirSync(logDir, { recursive: true });
    const date = new Date().toISOString().split('T')[0];
    this.logFile = path.join(logDir, `agent-${date}.jsonl`);
  }

  log(entry: Omit<AgentLogEntry, 'timestamp'>): void {
    const record: AgentLogEntry = {
      timestamp: new Date().toISOString(),
      ...entry,
    };
    fs.appendFileSync(this.logFile, JSON.stringify(record) + '\n');
  }

  logToolCall(params: {
    sessionId: string;
    agentName: string;
    toolName: string;
    inputSummary: string;
    outputSummary: string;
    durationMs: number;
    success: boolean;
    error?: string;
  }): void {
    this.log({
      session_id: params.sessionId,
      agent_name: params.agentName,
      event_type: 'tool_call',
      tool_name: params.toolName,
      input_summary: params.inputSummary,
      output_summary: params.outputSummary,
      duration_ms: params.durationMs,
      success: params.success,
      error: params.error,
    });
  }

  logLLMCall(params: {
    sessionId: string;
    agentName: string;
    model: string;
    tokenCount: number;
    durationMs: number;
    success: boolean;
  }): void {
    this.log({
      session_id: params.sessionId,
      agent_name: params.agentName,
      event_type: 'llm_call',
      tool_name: params.model,
      token_count: params.tokenCount,
      duration_ms: params.durationMs,
      success: params.success,
    });
  }
}
```

### 使用示例

```typescript
const logger = new AgentLogger('align-20240115-abc123');

const start = Date.now();
try {
  const result = await readRepoFile({ repo: 'app-home', path: 'src/Home.tsx' });
  logger.logToolCall({
    sessionId: 'align-20240115-abc123',
    agentName: 'research',
    toolName: 'read_repo_file',
    inputSummary: 'repo=app-home path=src/Home.tsx',
    outputSummary: `${result.length} chars`,
    durationMs: Date.now() - start,
    success: true,
  });
} catch (err) {
  logger.logToolCall({
    sessionId: 'align-20240115-abc123',
    agentName: 'research',
    toolName: 'read_repo_file',
    inputSummary: 'repo=app-home path=src/Home.tsx',
    outputSummary: '',
    durationMs: Date.now() - start,
    success: false,
    error: String(err),
  });
}
```

## 3. 追踪的实现

追踪的核心是用 `session_id + step_id` 串联同一次任务的所有操作。不需要引入 OpenTelemetry 等复杂框架，一个简单的 `Tracer` 类就够了：

```typescript
class SimpleTracer {
  private sessionId: string;
  private steps: Array<{
    stepId: number;
    event: string;
    durationMs: number;
    success: boolean;
    detail?: string;
  }> = [];
  private stepCounter = 0;

  constructor(sessionId: string) {
    this.sessionId = sessionId;
  }

  async trace<T>(
    event: string,
    fn: () => Promise<T>
  ): Promise<T> {
    const stepId = ++this.stepCounter;
    const start = Date.now();
    try {
      const result = await fn();
      this.steps.push({
        stepId,
        event,
        durationMs: Date.now() - start,
        success: true,
      });
      return result;
    } catch (err) {
      this.steps.push({
        stepId,
        event,
        durationMs: Date.now() - start,
        success: false,
        detail: String(err),
      });
      throw err;
    }
  }

  summary(): string {
    const lines = [`Session: ${this.sessionId}`];
    for (const step of this.steps) {
      const status = step.success ? '✓' : '✗';
      lines.push(`  ${status} step ${step.stepId}: ${step.event} (${step.durationMs}ms)`);
      if (step.detail) lines.push(`    └─ ${step.detail}`);
    }
    return lines.join('\n');
  }
}
```

使用时，把每个工具调用和 LLM 调用包在 `tracer.trace()` 里：

```typescript
const tracer = new SimpleTracer('align-20240115-abc123');

const memory = await tracer.trace('read_memory', () => readMemory());
const files = await tracer.trace('list_repo_files(app-home)', () =>
  listRepoFiles({ repo: 'app-home' })
);
const analysis = await tracer.trace('LLM:claude-sonnet', () =>
  callLLM({ model: 'claude-sonnet-4', ... })
);

console.log(tracer.summary());
// Session: align-20240115-abc123
//   ✓ step 1: read_memory (23ms)
//   ✓ step 2: list_repo_files(app-home) (156ms)
//   ✓ step 3: LLM:claude-sonnet (2341ms)
```

## 4. 关键 Dashboard 指标

当你有了结构化日志，就可以写脚本生成每日 Dashboard。追齐 Agent 的关键指标：

| 指标 | 含义 | 目标值 |
|------|------|--------|
| 每日追齐功能数 | 系统吞吐量 | 按需定 |
| 平均工具调用次数/功能 | 效率（越少越好） | < 15 次 |
| 平均耗时/功能 | 用户等待时间 | < 3 分钟 |
| 平均 token 消耗/功能 | 成本代理指标 | < 8000 tokens |
| 任务失败率 | 可靠性 | < 5% |
| 失败原因分布 | 指导改进优先级 | — |

每日摘要脚本示例（从 JSONL 聚合）：

```typescript
import * as fs from 'fs';

function generateDailySummary(logFile: string): void {
  const lines = fs.readFileSync(logFile, 'utf-8').trim().split('\n');
  const entries: AgentLogEntry[] = lines.map(l => JSON.parse(l));

  const sessions = new Map<string, AgentLogEntry[]>();
  for (const entry of entries) {
    if (!sessions.has(entry.session_id)) {
      sessions.set(entry.session_id, []);
    }
    sessions.get(entry.session_id)!.push(entry);
  }

  let totalTokens = 0;
  let totalToolCalls = 0;
  let failures = 0;

  for (const [sessionId, sessionEntries] of sessions) {
    const tokens = sessionEntries
      .filter(e => e.event_type === 'llm_call')
      .reduce((sum, e) => sum + (e.token_count ?? 0), 0);
    const toolCalls = sessionEntries.filter(e => e.event_type === 'tool_call').length;
    const failed = sessionEntries.some(e => !e.success);

    totalTokens += tokens;
    totalToolCalls += toolCalls;
    if (failed) failures++;
  }

  const sessionCount = sessions.size;
  console.log(`=== Daily Summary ===`);
  console.log(`Total sessions: ${sessionCount}`);
  console.log(`Failure rate: ${((failures / sessionCount) * 100).toFixed(1)}%`);
  console.log(`Avg tool calls/session: ${(totalToolCalls / sessionCount).toFixed(1)}`);
  console.log(`Avg tokens/session: ${(totalTokens / sessionCount).toFixed(0)}`);
}
```

> [!info] 最小可行可观测性
> 不要一开始就追求完整的 Dashboard。**最小可行可观测性**的路径是：
> 1. 第一步：结构化 JSONL 日志（1天工作量）
> 2. 第二步：每日摘要脚本（半天工作量）
> 3. 第三步：失败告警（邮件/Slack 通知）（半天工作量）
> 4. 第四步：可视化 Dashboard（等系统稳定后再做）
>
> 先有日志，再优化。复杂的 Dashboard 等你真的需要的时候再加。

## 本节小结

可观测性三层模型：

- **日志**：结构化 JSONL，每条记录包含 session_id、event_type、tool_name、duration_ms、success
- **追踪**：SimpleTracer 用 session_id 串联链路，不需要复杂框架
- **指标**：每日脚本从 JSONL 聚合，重点关注失败率、token 消耗、工具调用次数
