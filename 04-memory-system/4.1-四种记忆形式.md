# 4.1 四种记忆形式：Agent 的信息存储全景

> [!abstract] 本节导读
> "记忆"在 Agent 系统里不是一个单一的概念——它有四种形式，分布在不同的位置，有不同的特性。
> 理清这四种形式，是设计任何有状态 Agent 的前提。
> **预计时长**：核心阅读 20 分钟。

---

> [!question] 带着问题阅读
> 当一个 Agent 完成了今天的任务，明天再启动时，它"记得"什么？
> 这个"记得"是怎么实现的？

---

## 1. 四种记忆形式全景

![Agent 记忆类型|603](https://gw.alipayobjects.com/zos/k/7w/agent_memory_types.svg)

Agent 的记忆可以按两个维度来分类：
- **存储位置**：在上下文窗口内（In-Context）/ 在外部存储中（External）
- **持久性**：会话结束即消失 / 跨会话持久

这产生了四种记忆形式：

---

### 形式一：In-Context Storage（上下文内存储）

**位置**：messages[] 的 L4/L5 层
**持久性**：仅限当前会话，会话结束即消失
**特点**：LLM 随时可见，零检索延迟，但受上下文窗口大小限制

这就是第二章讲的工作记忆（L4）和对话流（L5）。
对于短任务，它是唯一需要的记忆形式。

**追齐 Agent 中的使用**：当前功能点的追齐进度、本次会话的分析中间结论。

---

### 形式二：External Storage（外部存储）

**位置**：文件系统、数据库、向量数据库
**持久性**：跨会话持久，除非主动删除
**特点**：需要通过工具读写，有检索延迟，但容量无限制

这是实现"长期记忆"的核心手段。
External Storage 本身又分两类：
- **结构化存储**：JSON 文件、数据库——精确查找
- **向量存储**：Embedding + 向量库——语义相似检索

**追齐 Agent 中的使用**：repo-map 配置、历史追齐结论、仓库摘要缓存。

---

### 形式三：In-Weights（模型权重内）

**位置**：模型参数本身（通过训练/微调写入）
**持久性**：永久（直到下次训练）
**特点**：LLM 天然"知道"的背景知识，不占上下文，但更新成本极高

这是模型预训练和微调时注入的知识。
对大多数 Agent 开发者来说，这层不在日常工程范围内（微调是专项工程）。

**追齐 Agent 中的使用**：LLM 对 React、TypeScript 的内置理解（不需要我们提供）。

---

### 形式四：In-Cache（缓存内）

**位置**：LLM 服务器端的 KV Cache
**持久性**：会话级（TTL 约 5 分钟，见 2.5）
**特点**：稳定前缀被缓存，重复调用免费复用

这就是第二章讲的 Prompt Caching。
它本质上也是一种"记忆"——把稳定内容缓存起来，后续调用不重复处理。

**追齐 Agent 中的使用**：System Prompt、工具定义、当前任务的 Skill 内容。

---

## 2. 四种形式的特性对比

| 形式 | 容量 | 检索 | 持久性 | 开发成本 |
|------|------|------|--------|---------|
| In-Context | 受窗口限制（~200K tokens）| 零延迟 | 会话级 | 极低 |
| External Storage | 无限 | 需要工具调用 | 永久 | 中 |
| In-Weights | 受训练数据限制 | 零延迟 | 永久 | 极高 |
| In-Cache | 等于上下文窗口 | 零延迟 | 会话级 | 低 |

> [!tip] 原则
> **绝大多数 Agent 只需要关注前两种：In-Context 和 External Storage。**
> In-Weights 需要微调，一般是专项工程；In-Cache 是框架自动处理的优化，按 2.5 节配置即可。
> 本章重点是 External Storage 的设计——什么值得存、怎么存、怎么读。

---

## 3. 记忆与上下文工程的关系

这两章看起来是不同的话题，但实际上有直接关联：

```
上下文工程（第二章）→ 管理"当前会话"的信息质量
记忆系统（第四章）→ 管理"跨会话"的信息积累

External Storage 是连接两者的桥：
- 会话结束时，把有价值的信息从 L4/L5 写入 External Storage（记忆巩固）
- 会话开始时，从 External Storage 读取相关记忆注入 L1/L3（记忆召回）
```

> [!example] 案例
> 功能追齐 Agent 第一次追齐购物车页面：
> - 会话结束时，把"购物车 H5 仓库的目录结构 + 技术栈"写入外部文件（记忆巩固）
> - 下次追齐购物车其他功能时，直接读取这份摘要（记忆召回），不需要重新探索仓库
> - 节省了约 4 次工具调用，上下文也更干净

---

## 本节小结

- 四种记忆：In-Context（当前会话）/ External（跨会话）/ In-Weights（训练权重）/ In-Cache（KV缓存）
- 日常 Agent 开发关注前两种；In-Cache 按第二章配置；In-Weights 是专项工程
- External Storage 是连接当前会话和跨会话的桥
