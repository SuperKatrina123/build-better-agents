## 1. 记忆系统全景

```
记忆形式选择：
  ✅ In-Context（L4 工作记忆）  → 当前任务状态
  ✅ External Storage（文件系统）→ 跨会话记忆
  ⏭ In-Weights（微调）          → 暂不需要
  ✅ In-Cache（Prompt Caching）  → 已在第二章配置

External Storage 读取模式：
  ✅ 常驻注入    → MEMORY.md 摘要 + 进度（~500 tokens）
  ✅ 工具检索    → 仓库结构详情 + 功能追齐结论（按需）
  ⏭ 向量检索    → 暂不需要（条目 < 100）
```

## 3. 记忆工具清单

追齐 Agent 的工具列表在第三章基础上增加两个记忆工具：

```typescript
// 原有 4 个工具（第三章）+ 新增 2 个记忆工具

{
  name: "read_memory",
  description: "读取已存储的仓库结构摘要或功能追齐历史结论。会话开始时 MEMORY.md 已自动注入，此工具用于读取更详细的记忆内容。",
  inputSchema: {
    type: "object",
    required: ["key"],
    properties: {
      key: {
        type: "string",
        enum: [
          "completed",                        // 追齐状态
          "repo/{仓库名}/structure",           // 仓库结构
          "repo/{仓库名}/known-issues",        // 已知坑点
          "feature/{功能名}"                   // 功能追齐结论
        ],
        description: "记忆键，使用 MEMORY.md 中 availableMemories 列出的键"
      }
    }
  }
},

{
  name: "write_memory",
  description: "将重要发现写入长期记忆。当你发现了关于仓库结构的重要信息、完成了一个功能的追齐、或者需要更正错误记忆时调用。",
  inputSchema: {
    type: "object",
    required: ["key", "content"],
    properties: {
      key: { type: "string", description: "记忆键" },
      content: { type: "string", description: "要记忆的内容（Markdown 格式，≤1000 tokens）" },
      mode: {
        type: "string",
        enum: ["overwrite", "append"],
        default: "overwrite",
        description: "overwrite=覆盖旧值（适合结构信息），append=追加（适合坑点记录）"
      }
    }
  }
}
```

## 4. 会话协议（写入 System Prompt）

```markdown
## 记忆管理协议

**会话开始**：已自动注入 MEMORY.md，请先阅读其中的追齐进度和仓库知识。

**执行任务前**：
- 检查 MEMORY.md 中该功能是否已完成（避免重复工作）
- 如有该仓库的 structure 记忆，调用 read_memory 获取，跳过目录探索

**发现新知识时**：
- 立即调用 write_memory 更新，不要等到任务结束
- 写入原则：只写跨会话有复用价值的事实，不写推理过程

**任务完成时**：
- 调用 write_memory("feature/{功能名}", ...) 记录追齐结论
- 调用 write_memory("completed", ..., mode="overwrite") 更新进度

**发现记忆错误时**：
- 工具结果优先于记忆内容
- 立即调用 write_memory 纠正，并注明：「已更新记忆：{key}」
```

## 6. 第四章总结

| 节次 | 核心概念 | 一句话要点 |
|------|---------|-----------|
| 4.1 | 四种记忆形式 | In-Context / External / In-Weights / In-Cache |
| 4.2 | 记忆巩固 | 会话结束前把高价值信息写入外部，防止下次从零开始 |
| 4.3 | 写入策略 | 稳定事实→Markdown，动态状态→JSON，原子写入 |
| 4.4 | 读取策略 | 常驻注入（小量高频）+ 工具检索（按需精准）|
| 4.5 | MEMORY.md 模式 | 可读、可版本控制、人机协作维护的记忆入口 |
| 4.6 | 设计决策 | 6 个工具（4+2）+ 会话协议 + 完整目录结构 |

> [!tip] 原则
> **记忆系统不是越复杂越好。**
> 文件系统 + MEMORY.md + 两个记忆工具，就能让追齐 Agent 真正"记住"跨会话的知识。
> 向量数据库、知识图谱——留到真的需要时再加。
